# Production Notes

## App Shell

Meal Signal is a private authenticated app. The root route ships as a static
Next.js App Router shell, then Firebase Auth and Firestore hydrate user-owned
data on the client.

The app is intentionally marked `noindex` through metadata and `robots.ts` so
meal and symptom tracking screens are not treated as public SEO surfaces.

## Metadata and Icons

Common metadata lives in `src/app/layout.tsx`.

- `NEXT_PUBLIC_SITE_URL` controls `metadataBase` for production absolute URLs.
  Production uses `https://meal.amritr.xyz`.
- `src/app/icon.tsx` and `src/app/apple-icon.tsx` generate PNG app icons.
- `src/app/favicon.ico` is generated by `npm run assets:favicon`.
- `src/app/manifest.ts` exposes installable web app metadata.
- `public/sw.js` registers the PWA shell cache and offline fallback.
- `public/offline.html` is served for offline navigations. It intentionally
  does not cache private meal or symptom records.

## Caching and Headers

`next.config.ts` sets baseline security headers for all routes and explicit
cache headers for static public assets, generated metadata routes, and PWA
assets. `sw.js` is revalidated on every request so updates are discovered
quickly.

Next.js still owns its hashed `/_next/static/*` caching. Do not override those
headers unless the hosting platform requires it.

## Error Handling

Route-level render failures are handled by `src/app/error.tsx`.

Expected client failures use `getErrorMessage` in `src/lib/errors.ts`, which
maps Firebase Auth, Functions, and Firestore errors to user-facing messages.
Firestore subscriptions accept explicit error callbacks so live update failures
are visible in the app shell.

## Verification

Use the documented Docker/OrbStack workflow so platform-specific optional
packages are installed for the container instead of the host Mac.

```bash
mkdir -p /private/tmp/meal-tracker-node-modules

docker run --rm \
  -v "$PWD:/app" \
  -v /private/tmp/meal-tracker-node-modules:/app/node_modules \
  -w /app \
  node:22-bookworm npm ci

docker run --rm \
  -v "$PWD:/app" \
  -v /private/tmp/meal-tracker-node-modules:/app/node_modules \
  -w /app \
  node:22-bookworm npm run lint

docker run --rm \
  -v "$PWD:/app" \
  -v /private/tmp/meal-tracker-node-modules:/app/node_modules \
  -w /app \
  node:22-bookworm npm run build
```
